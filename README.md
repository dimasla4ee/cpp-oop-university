# Задачи на С++ по курсу Объектно-ориентированное программирование. Часть 1.
1. Напишите программу, которая запрашивает у пользователя два целых числа. После ввода программа должна вычислить и вывести на экран их сумму, разность, произведение и частное.
2. Реализуйте программу для решения квадратного уравнения вида ax^2+bx+c=0. Коэффициенты a, b и c вводятся пользователем. Программа должна использовать как минимум две отдельные функции:
    - для вычисления дискриминанта
    - для нахождения корней.

    Также необходимо предусмотреть обработку случаев, когда:
    - Дискриминант больше нуля (два различных вещественных корня).
    - Дискриминант равен нулю (один вещественный корень).
    - Дискриминант меньше нуля (нет вещественных корней).
3. Напишите программу, которая принимает на вход число N, а затем N целых чисел. Задача — определить и вывести количество простых чисел среди введённой последовательности.
4. Напишите программу, которая находит N-ое по счёту простое число.
5. Найти все целые числа в заданном диапазоне [A, B], которые являются палиндромами одновременно в десятичной и двоичной системах счисления. Числа, которые в десятичной или двоичной системе счисления начинаются с нуля, не считаются палиндромами.
# Задачи на С++ по курсу Объектно-ориентированное программирование. Часть 2.
## Задача 1: Класс Point (Базовый класс)
Создайте класс Point, который представляет точку в двумерном пространстве. Класс должен иметь:
- Приватные поля для хранения координат x и y (целые числа).
- Конструктор по умолчанию, инициализирующий x и y нулями.
- Конструктор, принимающий x и y в качестве параметров.
- Геттеры для получения значений x и y.
- Сеттеры для установки значений x и y.
- Метод print() для вывода координат точки в формате (x, y).
- Метод distanceTo(const Point& other) для вычисления расстояния до другой точки.
## Задача 2: Класс Rectangle
Создайте класс Rectangle, который представляет прямоугольник. Прямоугольник определяется двумя точками: левой нижней (bottomLeft) и правой верхней (topRight).
- Приватные поля: два объекта класса Point (bottomLeft и topRight) из первой задачи.
- Конструктор, принимающий четыре целых числа: x1, y1, x2, y2 для инициализации bottomLeft и topRight. Убедитесь, что x1 < x2 и y1 < y2 (если условие не выполняется, скорректируйте координаты так, чтобы прямоугольник был корректным, например, x1 всегда меньше x2).
- Геттеры для получения bottomLeft и topRight.
- Сеттеры для изменения bottomLeft и topRight.
- Методы calculateArea() и calculatePerimeter() для вычисления площади и периметра прямоугольника.
- Метод print() для вывода координат обеих точек прямоугольника.
## Задача 3: Система управления сотрудниками (Наследование)
Разработайте иерархию классов для управления сотрудниками компании.

Базовый класс Employee:
- Приватные поля: id (целое число), name (строка), hireDate (строка, например, "YYYY-MM-DD").
- Конструктор, инициализирующий эти поля.
- Геттеры для получения всех полей
- Сеттеры для изменения всех полей.
- Виртуальный метод displayInfo() для вывода общей информации о сотруднике.

Производный класс SalariedEmployee (Сотрудник на окладе):
- Наследует от Employee.
- Дополнительное приватное поле monthlySalary (вещественное число).
- Конструктор, принимающий все необходимые данные, включая оклад.
- Переопределяет displayInfo() для вывода информации о сотруднике на окладе, включая оклад.
- Метод calculateAnnualSalary() для вычисления годового оклада.

Производный класс HourlyEmployee (Сотрудник с почасовой оплатой):
- Наследует от Employee.
- Дополнительные приватные поля: hourlyRate (вещественное число), hoursWorked (целое число).
- Конструктор, принимающий все необходимые данные, включая почасовую ставку и отработанные часы.
- Переопределяет displayInfo() для вывода информации о сотруднике с почасовой оплатой, включая ставку и часы.
- Метод calculateWeeklyPay() для вычисления недельной зарплаты (Рабочий день составляет 8 часов).
## Задача 4: Полиморфизм в системе управления задачами 
Разработайте систему управления задачами, используя полиморфизм.

Абстрактный базовый класс Task:
- Приватные поля: title (строка), description (строка), isCompleted (булево).
- Конструктор, инициализирующий эти поля.
- Геттеры для получения всех полей
- Сеттеры для изменения всех полей.
- Метод markAsCompleted() для изменения статуса задачи.
- Виртуальный метод virtual void displayTaskDetails() const = 0; для вывода деталей задачи.
- Виртуальный деструктор.

Производный класс SimpleTask:
- Наследует от Task.
- Реализует displayTaskDetails() для вывода только общих деталей задачи.

Производный класс DeadlineTask:
- Наследует от Task.
- Дополнительное приватное поле deadlineDate (строка).
- Конструктор, принимающий все необходимые данные, включая дедлайн.
- Реализует displayTaskDetails() для вывода общих деталей и дедлайна.

Производный класс PriorityTask:
- Наследует от Task.
- Дополнительное приватное поле priorityLevel (целое число, 1-5, где 1 - наивысший).
- Конструктор, принимающий все необходимые данные, включая уровень приоритета.
- Реализует displayTaskDetails() для вывода общих деталей и уровня приоритета.
## Задача 5: Система управления библиотекой
Разработайте простую систему управления библиотекой, которая позволяет управлять книгами и журналами.

Базовый абстрактный класс LibraryItem:
- Приватные поля: title (строка), publicationYear (целое число), itemID (строка).
- Конструктор, инициализирующий эти поля.
- Геттеры для получения всех полей
- Сеттеры для изменения всех полей.
- Виртуальный метод virtual void displayDetails() const = 0; для вывода специфичных деталей элемента.
- Виртуальный деструктор.

Производный класс Book:
- Наследует от LibraryItem.
- Дополнительное приватное поле author (строка), isbn (строка).
- Конструктор, инициализирующий все поля.
- Реализует displayDetails() для вывода информации о книге.

Производный класс Magazine:
- Наследует от LibraryItem.
- Дополнительное приватное поле issueNumber (целое число).
- Конструктор, инициализирующий все поля.
- Реализует displayDetails() для вывода информации о журнале.

Класс Library:
- Приватное поле: std::vector<LibraryItem*> items; для хранения всех элементов библиотеки.
- Метод addItem(LibraryItem* item) для добавления элемента в библиотеку.
- Метод displayAllItems() для вывода деталей всех элементов в библиотеке, используя полиморфизм.
- Метод findItem(const std::string& itemID) для поиска элемента по ID и возврата указателя на него (или nullptr, если не найден).
- Деструктор, который освобождает память, выделенную для всех LibraryItem*.
